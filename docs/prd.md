Product Requirement Document (PRD)

Aplikasi Mobile E-PKL SMK (Flutter + Supabase)

Versi Dokumen: 1.0
Tanggal: 16 Januari 2026
Status: Draft Awal
Target Platform: Android (Utama) & iOS

1. Latar Belakang & Tujuan

Sekolah Menengah Kejuruan (SMK) memerlukan sistem yang efisien untuk memantau kehadiran dan aktivitas siswa selama Praktik Kerja Lapangan (PKL). Sistem manual (kertas) rentan hilang dan sulit divalidasi. Aplikasi ini bertujuan mendigitalkan proses absensi (validasi lokasi) dan pelaporan harian siswa.

2. Tech Stack & Arsitektur

2.1 Backend & Infrastructure (Supabase)

Kita menggunakan pendekatan Backend-as-a-Service (BaaS) dengan Supabase.

Database: PostgreSQL (via Supabase).

Auth: Supabase Auth (Email & Password / NISN).

Storage: Supabase Storage (Untuk foto selfie & foto kegiatan).

API: REST / Realtime Subscriptions (Generated by Supabase).

2.2 Mobile Client (Flutter)

Framework: Flutter (Dart).

State Management: Riverpod (Disarankan) atau BLoC.

Maps Service: flutter_map (OpenStreetMap).

Location Service: geolocator.

Camera: image_picker atau camerawesome.

3. Fitur Utama (Functional Requirements)

3.1 Otentikasi Pengguna (Siswa)

Login: Siswa login menggunakan Email/NISN dan Password yang didaftarkan sekolah.

Validasi Akun: Sistem mengecek apakah siswa memiliki status aktif PKL.

Session: Token login disimpan lokal (Persistent Login).

3.2 Absensi Kehadiran (Geo-Tagging & Selfie)

Fitur Core untuk memvalidasi kehadiran siswa di lokasi industri.

Alur Logika:

Get Location: Aplikasi mengambil koordinat Lat/Long siswa saat ini.

Mock Location Check: Aplikasi mendeteksi apakah siswa menggunakan Fake GPS. Jika YA -> Blokir aksi.

Distance Calculation (Geofencing):

Hitung jarak antara User_Location vs Company_Location (Data dari DB).

Rumus: Haversine Formula.

Batas Toleransi (Radius): 50 - 100 meter.

Validasi Jarak:

Jika Jarak > Radius: Tampilkan Error "Anda berada di luar lokasi PKL".

Jika Jarak <= Radius: Buka Kamera.

Capture & Upload:

Ambil foto selfie (Wajib kamera depan).

Kompresi gambar.

Upload ke Supabase Storage -> bucket: attendances.

Insert record ke tabel attendance_logs.

3.3 Laporan Jurnal Kegiatan

Siswa mencatat aktivitas harian.

Formulir Jurnal:

Tanggal & Waktu: Otomatis (Current Timestamp).

Jenis Kegiatan: Input Teks / Dropdown (e.g., Perawatan, Instalasi, Administrasi).

Deskripsi: Textarea (Minimal 50 karakter).

Bukti Foto: Upload foto kegiatan (Kamera Belakang/Galeri).

Validasi: Jurnal hanya bisa diisi jika status absensi hari ini = Hadir.

3.4 Riwayat & Profil

Riwayat Absensi: List kehadiran bulan ini (Hadir/Telat/Alpha).

Riwayat Jurnal: List jurnal yang sudah dikirim statusnya (Pending/Approved).

5. Security Rules (Supabase RLS)

PENTING: Jangan lupa aktifkan Row Level Security (RLS).

Profiles: Select (Auth users), Update (Own Profile only).

Attendance & Journals:

Insert: Hanya user yang terautentikasi (auth.uid() = student_id).

Select: Siswa hanya bisa melihat datanya sendiri. Admin/Guru bisa melihat semua.

7. Kalkulasi Jarak (Snippet Logic)

Gunakan Distance().as(LengthUnit.Meter, ...) dari package latlong2.

// Pseudo-code logic
final Distance distance = Distance();
final double meter = distance.as(
    LengthUnit.Meter,
    LatLng(currentLat, currentLong),
    LatLng(companyLat, companyLong)
);

if (meter <= companyRadius) {
  allowCheckIn();
} else {
  showError("Jarak terlalu jauh: ${meter}m");
}
